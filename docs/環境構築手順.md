## 1. 初期状態で発生した問題と対応

### 1-1. NumPy バイナリ互換性エラー

**エラー内容**

```
ValueError: numpy.dtype size changed, may indicate binary incompatibility.
Expected 96 from C header, got 88 from PyObject
```

`xtcocotools` が NumPy 1.x 向けにビルドされているのに、NumPy 2.2.6 がインストールされていたため発生。

**対応**

```bash
pip install "numpy<2"
```

NumPy 1.26.4 がインストールされる。

---

### 1-2. OpenCV の NumPy 依存バージョン不整合

NumPy を 1.x に下げたことで、OpenCV 4.12 が `numpy>=2` を要求して競合。

**対応**

```bash
pip install "opencv-python<4.11" "opencv-contrib-python<4.11"
```

OpenCV 4.10.0.84 がインストールされる。

---

### 1-3. mmcv の C++ 拡張が見つからないエラー

**エラー内容**

```
ModuleNotFoundError: No module named 'mmcv._ext'
```

pip で入る `mmcv` はlite版（Python のみ）であり、`MultiScaleDeformableAttention` 等の C++ ops が含まれていない。

**対応方針**

OpenMMLab が配布するプリビルドホイール（C++ 拡張付き）を使う必要がある。
ただし、プリビルドホイールは特定の PyTorch バージョンごとにしか提供されていない。

調査の結果、Windows + Python 3.10 + CUDA 11.8 向けのプリビルドホイールが存在するのは以下の組み合わせ：

| PyTorch | mmcv プリビルド |
|---|---|
| 2.1.x | あり |
| 2.2.x | あり |
| 2.3.x | あり |
| 2.4.x | なし（win-cp310 向け） |
| 2.5 以降 | なし |

**対応手順**

#### Step 1: PyTorch を 2.1.0 にダウングレード

```bash
pip install torch==2.1.0 torchvision==0.16.0 --index-url https://download.pytorch.org/whl/cu118
```

#### Step 2: mmcv をアンインストールしてからプリビルド版を再インストール

```bash
pip uninstall mmcv -y
pip install mmcv==2.1.0 -f https://download.openmmlab.com/mmcv/dist/cu118/torch2.1/index.html --no-cache-dir
```

### 1-6. 起動.bat が処理完了後に即座に閉じる

`起動.bat` に `pause` がなかったため、処理完了と同時にコマンドプロンプトが閉じ、
正常終了なのかエラーなのか判別できなかった。

`起動.bat` の末尾に以下を追加：

```bat
echo.
echo === 処理完了 ===
pause
```

## 7. STEP3（単眼3D骨格検出）の補足

### 3Dバックエンドの切替

`configs/step3_pose3d.yaml` の `runtime.backend` で挙動を切り替える。

| backend | 入力 | 出力 | 用途 |
|---|---|---|---|
| `mmpose` (デフォルト) | `input.video_path`（動画） | COCO17 相対3Dキーポイント | 本番推論。MMPose の 3D lifter（MotionBERT 等）を動かす |
| `fallback` | `input.keypoints_jsonl`（STEP1の出力） | ダミー3D | 開発・テスト時。モデル重み不要 |

### MotionBERT 重みの自動ダウンロード

初回 `backend: mmpose` 実行時、MMPose が `human3d` エイリアスに紐づく MotionBERT 系モデルの重みを
OpenMMLab 配布先から自動ダウンロードする。ネットワーク接続が必要。

ダウンロード先（OSによる）:
- Windows: `%HOMEPATH%\.cache\torch\hub\checkpoints\`
- Linux/Mac: `~/.cache/torch/hub/checkpoints/`

### H36M → COCO17 の内部変換

MotionBERT の出力は H36M 17 関節順だが、STEP4 の指標計算と整合させるため、
`src/features/keypoint_convert.py:h36m17_to_coco17` で COCO17 順に変換している。
COCO の目・耳に対応する H36M 関節は存在しないため、その位置の XYZ は NaN となる。

### 関節定義を切り替えたい場合

`runtime.keypoint_convention: h36m17` を指定すると H36M 17 関節のまま出力される。
ただし STEP4 の `features/angles_3d.py`・`posture_metrics.py` は COCO17 インデックス前提のため、
STEP4 指標は期待通りにならない点に注意。
