# 環境構築手順（ArUco_MMPose_Motion）

このドキュメントは `src/main.py` の STEP A〜E パイプライン向けです。

## 1. セットアップ

```bash
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
python -m pip install -U pip setuptools wheel
pip install -r requirements.txt
```

> `torch/torchvision` は GPU/CUDA 構成に合わせて導入してください（例: cu118）。

### 1-1. PyTorch（CUDA 11.8 例）

```bash
pip install torch==2.1.0 torchvision==0.16.0 --index-url https://download.pytorch.org/whl/cu118
```

### 1-2. OpenMMLab 依存（例）

```bash
pip install mmengine==0.10.4
pip install mmcv==2.1.0 -f https://download.openmmlab.com/mmcv/dist/cu118/torch2.1/index.html --no-cache-dir
pip install mmdet==3.3.0 mmpose==1.3.2
```

### 1-3. OpenCV contrib（ArUco 必須）

```bash
pip uninstall -y opencv-python opencv-contrib-python
pip install opencv-contrib-python==4.10.0.84
```

## 2. 動作確認

```bash
python scripts/env_check.py
python -m pytest tests/ -v
```

## 3. 実行方法

```bash
python -m src.main --step a --config configs/step_a_aruco.yaml
python -m src.main --step b --config configs/step_b_pose2d.yaml
python -m src.main --step c --config configs/step_c_sync.yaml
python -m src.main --step d --config configs/step_d_triangulation.yaml
python -m src.main --step e --config configs/step_e_smoothing.yaml
```

`--step all` 実行時は各ステップのデフォルト config を利用します（`--config` 併用不可）。

## 4. よくある問題

### 4-1. `ImportError: libGL.so.1`

```bash
sudo apt-get update
sudo apt-get install -y libgl1 libglib2.0-0
```

### 4-2. `cv2.aruco` がない

`opencv-python` ではなく `opencv-contrib-python` を使用してください。

## 5. 出力ファイル命名

`src/common/output_naming.py` により、主要出力はタイムスタンプ付きファイル名で保存されます。
